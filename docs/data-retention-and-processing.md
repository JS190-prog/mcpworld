# 자료 처리 및 보관 정책 초안

작성일: 2026-06-18 KST

## 기본 원칙

MCP World SaaS는 로컬 파일 원문을 서버에 기본 보관하지 않는 구조를 우선한다.

## 자료 분류

| 구분 | 처리 방향 |
|---|---|
| 계정 정보 | 회원 관리 목적 범위에서 보관 |
| 결제 식별 정보 | 결제사 식별값과 구독 상태만 최소 보관 |
| 로컬 에이전트 상태 | 연결 여부와 최근 접속 시각 중심으로 보관 |
| 로컬 원본 파일 | 기본 비보관 |
| 업로드 자료 | 기능 제공 시 별도 동의와 보관 기간 고지 |
| 서비스 로그 | 보안과 장애 대응 목적의 최소 항목만 보관 |

## 보관 기간 초안

| 자료 | 보관 기준 |
|---|---|
| 계정 정보 | 회원 탈퇴 시까지, 법령상 보관 필요 시 해당 기간 |
| 결제 기록 | 관련 법령상 보관 기간 |
| 분쟁 처리 기록 | 관련 법령상 보관 기간 |
| 보안 로그 | 내부 정책상 정한 단기 기간 |
| 세션 정보 | 세션 종료 후 필요한 범위에서 단기 보관 |
| 업로드 자료 | 별도 동의한 기간까지 보관 후 삭제 |

## 파기 원칙

- 보관 목적이 끝난 자료는 지체 없이 파기한다.
- 전자 자료는 복구가 어렵도록 삭제한다.
- 종이 자료가 생기는 경우 분쇄 또는 소각한다.
- 파기 내역은 내부 관리 기록으로 남긴다.

## 보호 조치

- 전송 구간 보호
- 저장 구간 보호
- 접근 권한 제한
- 로그 마스킹
- 관리자 접근 기록 관리
- 공개 저장소에 설정값과 비밀값 업로드 금지
